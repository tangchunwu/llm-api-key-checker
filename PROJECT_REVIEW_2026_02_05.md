## 2. 三种玩法与命令大全 (保姆级)

我们现在支持三种玩法，想怎么玩都行。

### 玩法一：自己电脑玩 (本地开发)

适合用来修改代码、调试功能。

* **启动命令**: `npm run dev:full`
  * *解释*: 这相当于在你电脑上同时把"前台"(网页)和"后厨"(服务器)都跑起来。
* **关闭方法**: 在黑框框里按 `Ctrl + C`，或者关掉窗口。

### 玩法二：装在浏览器里玩 (插件版)

适合日常使用，最隐私，最方便。

* **构建命令**: `npm run build`
  * *解释*: 这就像把做好的菜打包成盒饭 (`dist` 目录)。如果改了代码，必须运行这个命令重新打包。
* **安装方法**: 浏览器 -> 扩展管理 -> 开发者模式 -> 加载 `dist` 文件夹。

### 玩法三：发布到网上玩 (Cloudflare 部署)

适合分享给朋友，生成一个永久网址。

* **登录命令**: `npx wrangler login`
  * *解释*: 第一次用需要登录。就像办一张"开店许可证"。
* **发布命令**: `npm run deploy`
  * *解释*: 一键把你的代码上传到云端。Cloudflare 会自动帮你托管网页和后台。

## 3. 十万个为什么 (通俗版)

### Q1: 为什么 Github Pages 这种免费空间不行，非要用 Cloudflare？

* **GitHub Pages (纯静态网页)**:
  * 就像**发传单**。你给别人一张菜单（网页），但他点菜时，后面**没有厨房**（后端服务器）。
  * 如果他想拿着菜单直接去**五星级酒店 (OpenAI)** 拿菜，门口的**保安 (浏览器跨域限制/CORS)** 会把他拦住：“不许进，你是外面的传单！”
  * *结局*: 点菜失败。
* **Cloudflare Workers**:
  * 就像**流动餐车**。你不仅给了菜单，还自带了一个**微型厨房**。
  * 客人点菜 -> 餐车厨房接单 -> 厨房去酒店拿菜（因为是内部渠道，保安不拦） -> 做好端给客人。
  * *结局*: 完美运行。

### Q2: 那为什么“插件版”连厨房可以不要？

* **插件版 (浏览器插件)**:
  * 这是持有**VIP特权卡**的内部人员。
  * 你在 `manifest.json` 里申请了特权，**保安 (浏览器)** 一看你有证，直接放行。
  * 所以你不需要厨房（服务器），直接拿着菜单就能去 OpenAI 拿数据。这是成本最低、最隐私的做法。

### Q3: 为什么项目文件有 180MB，发给别人只要 0.2MB？

* **180MB (`node_modules`) 是“装修队”**:
  * 里面有吊车、水泥搅拌机、脚手架... 这些是盖房子（写代码/打包）时必须要用的工具。
* **0.2MB (`dist`) 是“盖好的房子”**:
  * 这就是最终成品。
* **结论**: 你把房子送给朋友住（分享插件），**不需要把装修队和搅拌机也一起送过去**。
